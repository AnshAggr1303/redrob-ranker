# Redrob Intelligent Candidate Ranking System
> Hackathon submission for the Redrob India Runs Data & AI Challenge

## What this does
Ranks 100,000 candidates against a job description the way a great recruiter would — not by matching keywords, but by understanding who genuinely fits the role.

The system uses a two-phase pipeline:
- **Phase 1 (offline):** Converts all candidate narratives into vector embeddings, extracts structured signals, and flags impossible/honeypot profiles. Run once, no time limit.
- **Phase 2 (runtime):** Loads pre-computed artefacts and scores every candidate in under 5 minutes on a single CPU using five scoring layers — semantic similarity, structured signals, availability, platform engagement, and verified assessments.

## Reproduce the submission

### 1. Clone and set up environment
```bash
git clone https://github.com/AnshAggr1303/redrob-ranker.git
cd redrob-ranker

python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Add the dataset
Place the `candidates.jsonl` file (465 MB) inside `data/`:
```
redrob-ranker/data/candidates.jsonl
```
This file is not committed to the repo due to size.

### 3. Run Phase 1 — Pre-computation
```bash
python src/precompute.py \
  --candidates ./data/candidates.jsonl \
  --jd ./data/signal_jd.txt \
  --out ./artefacts
```

This generates three artefacts in `artefacts/`:
- `embeddings.npy` — ~140 MB, candidate narrative vectors
- `jd_embedding.npy` — JD signal vector
- `metadata.parquet` — structured scoring fields

**Expected runtime:** 10–30 minutes depending on hardware (see GPU note below).

#### Speed up with Apple Silicon MPS (recommended for Mac M1/M2/M3)
Check if your chip supports MPS:
```bash
python -c "import torch; print(torch.backends.mps.is_available())"
```
If this prints `True`, open `src/precompute.py` and add one line after `model = SentenceTransformer(EMBEDDING_MODEL)`:
```python
model = model.to("mps")
```
This moves computation to the Apple GPU and cuts runtime from ~2 hours to ~8 minutes.

### 4. Run Phase 2 — Ranking
```bash
python src/rank.py \
  --candidates ./data/candidates.jsonl \
  --out ./outputs/submission.csv
```

Must complete in under 5 minutes on CPU. Produces `outputs/submission.csv` with 100 ranked candidates.

## For contributors / teammates

If you are running `precompute.py` to generate the artefacts, send these three files back after it completes:
- `artefacts/embeddings.npy`
- `artefacts/jd_embedding.npy`
- `artefacts/metadata.parquet`

`honeypot_ids.txt` is already committed to the repo, no need to resend.

Place received files in the `artefacts/` folder before running `rank.py`.

## Scoring approach

| Layer | Signal | Weight |
|---|---|---|
| Semantic similarity | Cosine distance between candidate narrative and JD signal vector | 55% |
| Structured scoring | YOE band, location, notice period, consulting penalty, education tier | 30% |
| Availability multiplier | Last active date, open-to-work, response rate, interview completion | Multiplier |
| GitHub activity | Normalized open-source activity score | 5% |
| Skill assessments | Verified platform scores on relevant AI/ML skills | 5% |

Candidates are hard-disqualified before scoring if they are: honeypot profiles, consulting-only careers, completely irrelevant titles, or outside India and unwilling to relocate.
