# Redrob Intelligent Candidate Ranking System
> **Team 2bits** · Redrob India Runs Data & AI Challenge 2026  
> Ansh Agrawal · Atharva Tiwari

Ranks 100,000 candidates against a job description the way a great recruiter would — not by matching keywords, but by understanding who genuinely fits the role.

---

## How it works

A two-phase pipeline:

**Phase 1 — Offline pre-computation (run once, no time limit)**
- Detects honeypot profiles via temporal impossibility checks
- Extracts structured signals from all 100k candidates into parquet
- Encodes candidate narratives (headline + summary + career descriptions) using `all-MiniLM-L6-v2`
- Encodes the signal-only JD into a single vector

**Phase 2 — Runtime ranker (<5 min on CPU)**
Five scoring layers:
1. Hard disqualifiers — honeypots, consulting-only, irrelevant titles, overseas/no-relocation
2. Semantic score — cosine similarity between candidate narrative and JD vector (55%)
3. Structured score — YOE band, location, notice period, GitHub, education tier (30%)
4. Availability multiplier — last_active decay, open_to_work, response rate, interview completion
5. Final formula — `S_final = (S_sem×0.55 + S_struct×0.30) × avail_mult + github×0.05 + assess×0.05`

## Quick start

### 1. Clone and set up environment
```bash
git clone https://github.com/AnshAggr1303/redrob-ranker.git
cd redrob-ranker
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Add the dataset
Place `candidates.jsonl` (465 MB) inside `data/`:
```
redrob-ranker/data/candidates.jsonl
```

### 3. Run Phase 1 — Pre-computation
```bash
python src/precompute.py \
  --candidates ./data/candidates.jsonl \
  --jd ./data/signal_jd.txt \
  --out ./artefacts
```

**Apple Silicon (M1/M2/M3) speed-up:** If `python -c "import torch; print(torch.backends.mps.is_available())"` prints `True`, add `model = model.to("mps")` after `model = SentenceTransformer(EMBEDDING_MODEL)` in `precompute.py`. Cuts runtime from ~2hr to ~8min.

**Skip embeddings** (if only honeypot logic changed):
```bash
python src/precompute.py --skip-embeddings
```

### 4. Run Phase 2 — Ranking
```bash
python src/rank.py \
  --candidates ./data/candidates.jsonl \
  --out ./outputs/submission.csv
```

Completes in **~4 seconds on CPU**. Produces `outputs/submission.csv`.

### 5. Validate output
```bash
python validate_submission.py outputs/submission.csv
# Expected: Submission is valid.
```

## Run the Streamlit demo locally

```bash
pip install streamlit
streamlit run app/streamlit_app.py
```

Open `http://localhost:8501` in your browser.

**What it does:**
- Upload any JSON file of candidates (use `data/sample_candidates.json` to test)
- Runs the full five-layer ranking pipeline on the uploaded sample
- Shows ranked table with scores, titles, locations, and reasoning
- Download ranked CSV button included

**Requirements for the demo:**
- `artefacts/jd_embedding.npy` must exist (generated by precompute.py)
- OR `data/signal_jd.txt` must exist (app will re-encode it on first run)

## Scoring formula

```
S_final = (S_sem × 0.55 + S_struct × 0.30) × avail_mult
        + github_norm × 0.05
        + assess_bonus × 0.05
```

| Component | Weight | Signal |
|---|---|---|
| S_sem | 55% | Cosine similarity of career narrative vs JD vector |
| S_struct | 30% | YOE band, location, notice period, GitHub, education |
| avail_mult | multiplier | last_active decay, open_to_work, response rate, interview rate |
| github_norm | 5% | github_activity_score / 100 |
| assess_bonus | 5% | +0.02 if any relevant AI/ML assessment ≥ 70 |
